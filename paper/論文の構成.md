## Research Question

How can open agent communication, on-chain identity, and protocol-level micropayments be integrated into a cohesive marketplace architecture with LLM-based orchestration, so that a user's high-level task is autonomously decomposed into agent discovery, delegation, and payment across organizational boundaries?

（オープンなエージェント通信プロトコル、オンチェーン ID、プロトコルレベルのマイクロペイメントを、LLM オーケストレーション付きマーケットプレイスとして統合し、ユーザーの高レベルタスクを組織横断的なエージェント発見・委任・決済へ自律的に分解できるか？）

### Contributions（tex 反映済）

1. A2A・x402・ERC-8004 をオーケストレーション層で統合したマーケットプレイスアーキテクチャ（3層設計）
2. ReAct エージェントによるタスク分析 → 発見 → 仕様確認 → 有償実行の自律パイプライン（`discover_agents` / `fetch_agent_spec` / `execute_and_evaluate_agent`）
3. Base Sepolia 上の動作プロトタイプと E2E デモ（flight / hotel-search agents）

## Vaziry et al. (arXiv:2507.19550) との差別化

| 観点 | Vaziry et al. | UniAgent |
|------|--------------|----------|
| 位置づけ | プロトコル層の実現可能性検証 | プロトコル統合マーケットプレイスアーキテクチャ |
| AgentCard | 個別スマートコントラクト | ERC-8004 標準レジストリ |
| 発見 | 4方式提案（未統合） | レジストリ + Webhook キャッシュ |
| 決済 | x402 + A2A の PoC | x402 + Privy 委譲署名（EIP-3009） |
| オーケストレーション | なし（A→B 固定） | LLM ReAct（discover → fetch\_spec → execute） |
| ユーザー体験 | プロトコル PoC | チャット UI + SSE ストリーミング |
| 評価・信頼 | tx 頻度の間接推定を提案 | EAS attestation（設計言及、大規模評価は今後） |

**差別化の核**: Vaziry はプロトコル要素の技術的実現可能性を示した。UniAgent はそれらを統合し、LLM オーケストレーションでユーザータスクから E2E 自律取引を実現するマーケットプレイスを設計・実装した。

## セクション別メモ

### 1. Introduction

- 背景: LLM エージェント → マルチエージェント委任の需要
- **課題1**: A2A は通信を標準化するが発見は仕様外
- **課題2**: Vaziry et al. はプロトコル PoC のみ、E2E マーケットプレイス未設計
- **課題3**: GPT Store 等はプラットフォーム閉域（Frieden 2016 引用）
- Research Question / Contributions / 論文構成の案内
- **Teaser なし** — 最初の図は §4 の Fig.1

### 2. Background

- **A2A**: AgentCard、JSON-RPC、発見はアプリ依存
- **x402**: HTTP 402、EIP-3009、facilitator — §4.3 Fig.3 で UniAgent 版シーケンス参照
- **ERC-8004**: オンチェーン ID、Agent Card の CID 参照、staking
- **ReAct + LangChain**: オーケストレーション層の根拠

### 3. Related Work

- **Agent communication**: Ehtesham et al. survey（MCP vs A2A/ANP）— MCP は orchestrator 内部のみ
- **Agent payment**: x402 のみ（L402 / Skyfire / PaymanAI は tex に未記載）
- **Decentralized identity + Vaziry**: ERC-8004 集約 vs 個別 SC
- **Table 1** (`tab:vaziry`): Vaziry との明示比較
- **AutoGen** (Wu et al.): アプリ内 HITL — UniAgent はマーケットプレイス層が対象
- **GPT Store** + Frieden: 中央集権 vs オープン再実装

### 4. System Architecture

#### 4.1 Marketplace Layer

- ERC-8004 `AgentIdentityRegistry` が Single Source of Truth
- Alchemy Webhook → PostgreSQL キャッシュ → マーケットプレイス UI
- **Fig.2** 開発者フロー — §4.1 説明の直後に配置

#### 4.2 Orchestration Layer

- ReAct + LangChain、**3ツール**:
  - `discover_agents` — キャッシュから候補 Top3
  - `fetch_agent_spec` — Agent Card + OpenAPI を無課金取得（実行前の仕様確認）
  - `execute_and_evaluate_agent` — A2A + x402 + tx 検証 + EAS attestation
- **Orchestration loop**: タスク分解 → discover → fetch（複数候補）→ execute → 繰り返し（§4.2 本文 + Fig.5 参照）
- **発見ランキング**（触れる程度）: reliability / quality / deposit / freshness の composite score + ε-Greedy で Top3。式・係数は書かない

#### 4.3 Payment Layer

- **Fig.3** `x402-sequencial.png`: UniAgent 版 x402 シーケンス
  - Agent Service → External A2A Agent（402 + PAYMENT-REQUIRED）
  - Privy 委譲 EIP-3009 署名 → X-PAYMENT 再送
  - Facilitator verify / settle → Base Sepolia USDC
  - HTTP 200 + PAYMENT-RESPONSE
  - 注: EAS attestation は off-chain（図中は注記のみ）
- **EAS**（触れる程度）: 実行後 attestation → 将来の発見スコア入力。大規模評価は今後

#### 4.4 Integration Design

- §4 本文: Privy 認証 → SSE → ReAct ループ → 各層連携の価値
- **Fig.4** user-flow — §4.4 説明の直後に配置
